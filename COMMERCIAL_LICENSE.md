## Two-Layer Model

| Layer | What it is | License |
|-------|------------|---------|
| **Harness** | SDK, CLI, REST, MCP, UI, storage, exports, integrations | MIT |
| **Kernel** | Compiled scoring, branching, methodology synthesis | [Kernel License](legal/KERNEL_LICENSE.md) |

See [docs/kernel-boundary.md](docs/kernel-boundary.md) for the technical boundary.

## Kernel Boundary

The bundled compiled kernel runtime is the implementation boundary for:

- methodology synthesis
- candidate branching
- candidate scoring and ranking
- kernel-side review signals before human judgment
- domain or customer-specific kernel behavior (enterprise)

Application code should call the Pheo SDK, CLI, MCP, or REST surfaces. Do not add public fallback implementations for kernel behavior in the harness.
